二. 知识讲解

1.1.1. 函数的递归调用的概念

  • 什么是函数递归调用?

函数自己调用自己，这种调用称为“递归”调用，这样的函数称为“递归函数”。

举个有趣的例子：
从前有座山，山里有座庙，庙里有个老和尚对小和尚说：“从前有座山，山里有座庙，庙里有个老和尚对小和尚说：“从前有座山，山里有座庙......” ”

执行递归函数将反复调用其自身，每调用一次就进入新的一层，当最内层的函数执行完毕后，再一层一层地由里到外退出。

2.2.2. 函数的递归调用的理解

下面我们通过一个求阶乘的例子，看看递归函数到底是如何运作的。
阶乘 n!n!n! 的计算公式如下：
n!={1(n=0,1)n∗(n−1)!n>1n! = \begin{cases} 1&(n=0,1) \\ n * (n - 1)! & n>1 \end{cases}n!={1n∗(n−1)!​(n=0,1)n>1​
根据公式编写以下代码

#include <bits/stdc++.h>
using namespace std;

//求n的阶乘
long factorial(int n) {
    if (n == 0 || n == 1) {
        return 1;
    }
    else {
        return factorial(n - 1) * n;  // 递归调用
    }
}

int main() {
    int a;
    cin >> a;
    printf("Factorial(%d) = %ld\n", a, factorial(a));
    return 0;
}

输入: 5↙
运行结果

Factorial(5) = 120

factorial()factorial()factorial() 就是一个典型的递归函数。调用 factorial()factorial()factorial() 后即进入函数体，只有当 n==0n==0n==0 或 n==1n==1n==1 时函数才会执行结束，否则就一直调用它自身。

由于每次调用的实参为 n−1n-1n−1 ，即把 n−1n-1n−1 的值赋给形参 nnn ，所以每次递归实参的值都减 111 ，直到最后 n−1n-1n−1 的值为 111 时再作递归调用，形参 nnn 的值也为 111 ，递归就终止了，会逐层退出。

  • 要想理解递归函数，重点是理解它是如何逐层进入，又是如何逐层退出的，下面我们以 5! 为例进行讲解。
递归的进入

(1)(1)(1) 求 5!5!5! ，即调用 factorial(5)factorial(5)factorial(5) 。当进入 factorial()factorial()factorial() 函数体后，由于形参 n 的值为 555 ，不等于 000 或 111 ，所以执行 factorial(n−1)∗nfactorial(n-1) * nfactorial(n−1)∗n，也即执行factorial(4)∗5factorial(4) * 5factorial(4)∗5 。为了求得这个表达式的结果，必须先调用 factorial(4)factorial(4)factorial(4)，并暂停其他操作。换句话说，在得到 factorial(4)factorial(4)factorial(4) 的结果之前，不能进行其他操作。这就是第一次递归。

(2)(2)(2) 调用 factorial(4)factorial(4)factorial(4) 时，实参为 444 ，形参 nnn 也为 444 ，不等于 000 或 111，会继续执行 factorial(n−1)∗nfactorial(n-1) * nfactorial(n−1)∗n ，也即执行 factorial(3)∗4factorial(3) * 4factorial(3)∗4 。为了求得这个表达式的结果，又必须先调用 factorial(3)factorial(3)factorial(3)。这就是第二次递归。

(3)(3)(3) 以此类推，进行四次递归调用后，实参的值为 111 ，会调用 factorial(1)factorial(1)factorial(1)。此时能够直接得到常量 111 的值，并把结果 return ，就不需要再次调用 factorial()factorial()factorial() 函数了，递归就结束了。

递归的退出

当递归进入到最内层的时候，递归就结束了，就开始逐层退出了，也就是逐层执行 return 语句。

(1)(1)(1) nnn 的值为 111 时达到最内层，此时 return 出去的结果为 111，也即 factorial(1)factorial(1)factorial(1) 的调用结果为 111。

(2)(2)(2) 有了 factorial(1)factorial(1)factorial(1) 的结果，就可以返回上一层计算 factorial(1)∗2factorial(1) * 2factorial(1)∗2 的值了。此时得到的值为 222，return 出去的结果也为 222 ，也即 factorial(2)factorial(2)factorial(2) 的调用结果为 222。

(3)(3)(3) 以此类推，当得到 factorial(4)factorial(4)factorial(4) 的调用结果后，就可以返回最顶层。经计算，factorial(4)factorial(4)factorial(4) 的结果为 242424，那么表达式 factorial(4)∗5factorial(4) * 5factorial(4)∗5 的结果为 120120120 ，此时 return 得到的结果也为 120120120 ，也即 factorial(5)factorial(5)factorial(5) 的调用结果为 120120120，这样就得到了 5!5!5! 的值。

递归的条件

每一个递归函数都应该只进行有限次的递归调用，否则它就会进入死胡同，永远也不能退出了，这样的程序是没有意义的。

要想让递归函数逐层进入再逐层退出，需要解决两个方面的问题：

  • 存在限制条件，当符合这个条件时递归便不再继续。对于 factorial()factorial()factorial() ，当形参 nnn 等于 000 或 111 时，递归就结束了。
  • 每次递归调用之后越来越接近这个限制条件。对于 factorial()，每次递归调用的实参为 n−1n - 1n−1 ，这会使得形参 nnn 的值逐渐减小，越来越趋近于 111 或 000。

3.3.3. 函数的递归调用的应用

一个问题要想用递归的方法（函数）来解决，必须要符合两个条件。
(1)(1)(1) 可以把这个问题转化成一个新问题，而新问题的解法和原问题的解法完全相同，只是问题规模变小了；
(2)(2)(2) 必须要有一个明确的递归结束条件（递归边界）。

这与递归的条件相对应，这也是我们要将问题抽象为递归模型的关键。
这种抽象的思维能力需要我们进行一定的练习才能够适应、强化。
